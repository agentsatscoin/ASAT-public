# ASAT Roadmap

This roadmap is intentionally high-level.

It is designed to show real direction without exposing private implementation details.

---

## Phase 1 — Live protocol surface

Focus:
- public protocol presence
- operator direction
- proof-of-task framing
- trustable product positioning

Targets:
- live website
- clear operator narrative
- registry/task surface direction
- public proof of ongoing development

---

## Phase 2 — Operator loop hardening

Focus:
- cleaner operator path
- clearer proof flows
- stronger task/reward clarity
- better public confidence in system reality

Targets:
- stronger review surfaces
- clearer participation signals
- cleaner reward-state communication
- more disciplined UX around operator actions

---

## Phase 3 — Admin and trust hardening

Focus:
- stronger administrative trust model
- guardian-gated operations
- clearer multi-party security boundaries
- audit visibility

Targets:
- mature passkey-based admin flow
- hardened session model
- quorum-secured critical actions
- stronger operational review discipline

---

## Phase 4 — Trust and reputation expansion

Focus:
- deeper operator trust surfaces
- stronger reputation logic
- better proof quality signaling

Targets:
- richer participation history
- clearer operator quality signals
- more robust review/reward logic
- stronger trust compounding

---

## Phase 5 — Network coordination maturity

Focus:
- broader protocol coordination
- more expressive task classes
- stronger system-level trust surfaces

Targets:
- expanded operator roles
- broader coordination primitives
- more durable public proof surfaces
- stronger long-term protocol credibility

---

## Directional priorities

The roadmap is guided by these priorities:

1. real execution over empty narrative
2. clear trust over vague hype
3. secure control over convenience shortcuts
4. visible proof over hand-wavy claims
5. disciplined public exposure over code leakage

---

## What this roadmap is not

This is not a full implementation log.
This is not a release engineering document.
This is not a disclosure of private internal work.

It is a public direction map.
